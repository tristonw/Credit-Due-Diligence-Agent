## 六大核心模块

### 1. 上下文压缩系统
**核心问题**: 尽调材料太多，LLM 装不下  
**解决思路**: 三层压缩架构 + 结构化提取  
**关键创新**:
- 财务数据 → 健康度卡片
- 行业报告 → 定位卡片
- 舆情信息 → 情感摘要

[查看详细设计 →](modules/01-context-compression.md)

---

### 2. 会话管理系统
**核心问题**: 尽调是多阶段流程，需要状态管理  
**解决思路**: 状态机 + 持久化 + 恢复机制  
**关键创新**:
- 6 阶段状态机 (INIT → DATA_COLLECTION → ANALYSIS → DRAFT → REVIEW → FINAL)
- 自动保存 + 中断恢复
- 检查点机制

[查看详细设计 →](modules/02-session-management.md)

---

### 3. 记忆系统
**核心问题**: Agent 如何从历史尽调中学习？  
**解决思路**: 四层记忆 + 多维度索引  
**关键创新**:
- L1 瞬时记忆 → L4 长期记忆
- 案例记忆 + 教训记忆 + 模板记忆
- 实体/主题/时间/语义多维度索引

[查看详细设计 →](modules/03-memory-system.md)

---

### 4. 专家知识蒸馏
**核心问题**: 如何让 Agent 写得像行业专家？  
**解决思路**: 学习专家报告，提取模式，内化知识  
**关键创新**:
- 词汇知识提取
- 论证结构学习
- 深度标记识别
- Prompt 动态优化

[查看详细设计 →](modules/04-expert-distillation.md)

---

### 5. Agent 自我迭代
**核心问题**: Agent 如何持续进化？  
**解决思路**: 执行 → 评估 → 反思 → 进化 循环  
**关键创新**:
- 多维度评估 (完整性/准确性/深度/专业度/效率)
- 自动反思教训
- 版本管理 + 可回滚

[查看详细设计 →](modules/05-agent-iteration.md)

---

### 6. 企业知识管理 ⭐
**核心问题**: 如何让 Agent 真正"懂"企业？  
**解决思路**: 构建企业知识图谱，让 AI 能读懂  
**关键创新**:
- 知识图谱: 人/公司/合同/事件/关系
- 文档智能: 财报解析/合同理解/舆情监控
- 数据融合: 内部 + 外部 + 实时
- API 服务: GraphQL + 语义搜索

**受 Karpathy AI 知识库理念启发**  
[查看详细设计 →](modules/06-enterprise-knowledge.md)

## 核心设计原则

1. **单 Agent 架构** - 避免多 Agent 协调复杂性
2. **极致上下文压缩** - 在有限 Token 内处理海量信息
3. **分层记忆管理** - 快速检索历史经验
4. **自我迭代进化** - 越用越像行业专家
5. **Harness Engineering** - 精心设计的控制流和反馈循环

## 技术栈建议

| 层级 | 技术 |
|------|------|
| Agent Core | OpenClaw ACP |
| Context Management | 自研分层压缩 |
| Memory Store | SQLite + ChromaDB |
| Knowledge Graph | Neo4j / Dgraph |
| Vector Search | ChromaDB / Pinecone |
| Document Parsing | Python + LLM |
| API | GraphQL + FastAPI |
| Report Generation | python-docx + Jinja2 |
